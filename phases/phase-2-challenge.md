# Phase 2: 对抗质疑

## 概述

Phase 2 是 Teamflow 的核心创新阶段，目的是**让 Phase 1 的结论互相碰撞，在裂缝中诞生真正的洞见**。

## 为什么需要 Phase 2

平面并行的致命缺陷：三个 agent 各自得出结论，然后"礼貌性整合"。没有真正的交锋，就没有超越各自视角的洞见。

Phase 2 的价值：**让结论互相碰撞，迫使每个结论暴露自己的脆弱点**。

## 启用条件（量化）

### 必须启用（满足任一）
- Phase 1 有 3 个或以上 agent 输出
- 任务涉及多个专业领域（每个领域有不同结论）
- Phase 1 输出中存在明显矛盾或分歧
- 任务涉及高风险决策（影响用户生产环境）

### 可选启用
- 用户明确要求"深度交锋"或 `--challenge` 参数
- 任务复杂度高于阈值（子任务数 ≥ 5）
- 时间敏感任务（用户标注"重要"或"紧急"）

### 可跳过
- 任务只有 1-2 个子任务
- Phase 1 输出高度一致（无明显分歧）
- 时间敏感任务（快速交付优先）

## 执行流程

```
Phase 1 输出
    ↓
Challenger 读取所有 Phase 1 输出
    ↓
Challenger 提出质疑（必须引用具体内容）
    ↓
被挑战的 Worker 回应
    ↓
主 agent 汇总所有交锋
```

## Challenger Prompt 模板

```
你是对抗性思考专家。你的任务是质疑其他 agent 的结论。

原始任务：{user_task}

Phase 1 结论：
{agent_a_conclusion}
{agent_b_conclusion}
{agent_c_conclusion}

你的任务：
1. 找出每个结论中最脆弱的假设
2. 提出尖锐的质疑，不是为了否定，而是为了让思考更深入
3. 识别结论之间的矛盾或未覆盖的区域

### 质疑要求
1. **必须引用 Phase 1 输出的具体内容**
   - 不能泛泛而谈，必须引用原话
   - 例如："你得出'系统性能优秀'的结论，但报告中说平均响应时间是 800ms，请解释这个矛盾"

2. **指出该内容的前提假设**
   - 明确说出假设是什么
   - 例如："你假设所有用户都有高速网络，但未考虑移动用户的弱网环境"

3. **说明该假设可能失败的条件**
   - 具体描述边界情况
   - 例如："当并发用户超过 1000 时，这个假设会失效"

4. **提出替代方案或遗漏的维度**
   - 给出具体建议
   - 例如："建议补充弱网环境下的性能测试"

### 禁止的行为
- ❌ 泛泛而谈的"假设前提"
- ❌ 没有具体针对性的"潜在风险"
- ❌ 只说"有问题"而不说具体是什么问题

质疑格式：
## 对 @worker-a 的质疑
[必须引用 worker-a 的具体输出内容]

## 对 @worker-b 的质疑
[必须引用 worker-b 的具体输出内容]

## 对 @worker-c 的质疑
[必须引用 worker-c 的具体输出内容]

## 结论间的矛盾
[如果有的话]
```

## Worker 回应要求

当 Worker 被 Challenger 质疑时：

1. **必须回应**：不接受沉默
2. **必须引用**：引用被质疑的具体内容
3. **必须说明**：为什么假设成立，或承认漏洞
4. **可选修正**：如果质疑有效，可以修正结论

### 回应格式

```markdown
## 对 @challenger 质疑的回应

### 质疑 1：[引用 challenger 的具体质疑]
**回应**：...

### 质疑 2：[引用 challenger 的具体质疑]
**回应**：...
```

## 超时处理

| 场景 | 超时阈值 | 超时后动作 |
|------|----------|------------|
| Phase 2 Challenger | 3 分钟 | 跳过 Phase 2，直接进入收敛 |
| Worker 回应 | 2 分钟/每个 | 强制进入收敛，未回应视为认可质疑 |

## 输出要求

Phase 2 完成时，应输出：

1. **Challenger 质疑清单**：每个 Phase 1 输出的具体质疑
2. **Worker 回应清单**：每个质疑的回应
3. **结论修正**（如有）：Phase 1 结论的更新

## 与 Phase 3 的衔接

Phase 2 输出将传递给 Phase 3（收敛整合）：

```
Phase 2 输出
    ↓
Phase 3 Reviewer 读取
    ↓
Reviewer 审查所有交锋
    ↓
主 agent 综合洞见
```

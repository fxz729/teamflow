# Phase 3: 收敛整合

## 概述

Phase 3 是 Teamflow 工作流的最终阶段，核心目标是**综合所有交锋，输出超越原始视角的洞见**。

## 执行流程

```
Phase 1 输出 + Phase 2 交锋结果
    ↓
Reviewer 最终审查
    ↓
主 agent 综合洞见
    ↓
输出最终结论
```

## Reviewer Prompt 模板

```
你是审查专家。审查团队执行结果：

任务：{original_task}
任务类型：{B/C}

Phase 1 输出：
{worker_outputs}

Phase 2 质疑（如果有）：
{challenger_output}

审查要点：
1. 是否满足原始需求
2. 子任务间是否有冲突
3. 推理链条是否完整
4. Phase 2 的质疑是否得到了回应

输出格式：
## 审查结果
✅ 通过 / ⚠️ 需要补充 / ❌ 需要修改

## 整合洞察
[将所有输出综合成超越原始视角的核心洞见]

## 推理链条摘要
[从原始问题到最终洞见的完整推理路径]
```

## 最终输出格式

### Team 模式输出

```markdown
## Team Execution Summary

### 任务类型
[A 类：开放性思辨 / B 类：信息整合 / C 类：可执行规划]

### 执行阶段
- [x] Phase 1: @worker-a / @worker-b / @worker-c（并行探索）
- [x] Phase 2: @challenger（对抗质疑，可选）
- [x] Phase 3: 收敛整合

### Phase 1 输出摘要
| Worker | 核心推理路径 | 关键洞察 |
|--------|-------------|----------|
| @worker-a | [推理链条摘要] | [洞察1] |
| @worker-b | [推理链条摘要] | [洞察2] |
| @worker-c | [推理链条摘要] | [洞察3] |

### Phase 2 质疑与回应（如果有）
| 质疑点 | 来源 | 回应摘要 |
|--------|------|----------|
| [质疑1] | @challenger | [回应1] |
| [质疑2] | @challenger | [回应2] |

### 最终结论
---
**Reviewer 最终整合**：

[超越原始视角的核心洞见，包括：]
- [最终决策或结论]
- [推理链条摘要：从原始问题到最终洞见的完整路径]
- [决策建议：如果任务需要进一步行动，给出具体建议]

[@reviewer 对最终结论的质量审查，包括：]
- [✅ 通过 / ⚠️ 需要补充 / ❌ 需要修改]
- [对推理链条完整性的评估]
- [对潜在问题的提示]
```

## Reviewer 质量检查清单

在输出最终结论前，Reviewer 应确认：

- [ ] 所有 Phase 1 子任务都已完成或明确标记未完成
- [ ] Phase 2 的所有质疑都有回应（或明确说明跳过原因）
- [ ] 最终结论与原始需求对齐
- [ ] 推理链条完整：从问题 → 探索 → 质疑 → 综合
- [ ] 没有引入新的、未经验证的假设

## 主 agent 综合洞见的要求

综合洞见应：
1. **超越原始视角**：不是简单汇总，而是真正的新洞察
2. **保留推理链条**：让读者理解决策是如何得出的
3. **明确不确定性**：哪些是确定的，哪些还有疑问
4. **给出行动建议**：如果需要进一步行动，明确建议
